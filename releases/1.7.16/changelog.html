AI Assets Scanner 1.7.16

Released June 2, 2026.

Fixed — external target-stack probe robustness

  • The external target-stack probe now sends a browser Accept header. Some origin WAFs reject a request that has no Accept header with HTTP 415; the probe now gets a usable response instead of a rejection.
  • A rejected or errored probe (any 4xx) is now reported as “probe failed” instead of being mislabeled “not WordPress”, so a blocked probe no longer silently suppresses the optimizer bypass on a real WordPress target.
  • Negative / indeterminate probe verdicts are cached for 15 minutes (instead of 24 hours), so a transient block — a rate limit or a bot challenge — self-heals on the next scan rather than being pinned for a day.

Same-site scans are unaffected: they detect optimizers via local plugin introspection, not the external HTTP probe.
